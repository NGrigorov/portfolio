Neutron Charge

  • Code
  • WEbgl player
  • Itch.io

Video

This is an indie project done for a 7 day game-jam competition. Written entirely in C# via Unity3D. You control a particle and you destroy other particles. Some of them try to destroy you instead by either chasing/ramming or shooting you. It is a quick arcade-type game in which you have to get a high-score by collecting all the non-destrucive particles. As the game progresses the colors change makingg the player memorize what colors are the enemy to know which of the new colors is now the destructive one.

Maze generator

  • webgl player

Video

A maze generator developed in Unity3D that I eventually expanded into a simple dungeon generator. It uses a semi-random DFS algorithm that I customized to create any amount of main rooms, open areas, long corridoors and size of the dungeon itself. I am using this on an indie dungeon crawler rougelike game that I am developing in my free time.

GPU Instancing Optimization

Video

A small project that I used to understand how to efficiently place a lot of triangles in a scene while still keeping it playable. The scene consists of large trees which have a separate sprite for each individual leaf. There are about 1 million triangles in view at any given time. Only objects in the camera path are rendered to further increase performance.

Ray-Tracing engine

  • code

Currently working from time to time on a self-built ray-tracing engine in C#. Excluding the OpenTK renderer, everything is hand-written, including the math class. It currently runs on the CPU so its not too efficient. The ray-tracer is still in early stage so some of the important parameters are missing such as Global Illumination, transparent Refraction and further optimization by transfering the code to the GPU. The user can move the camera around and each frame the viewport is recalculated from the new camera position to display the scene from that POV. The ray tracer currently has: path-tracing, reflection, non-transparent refraction.
